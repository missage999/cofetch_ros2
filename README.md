# Cofetch ROS2 项目架构

## 1. 项目和概述

### 1.1 项目简介
Cofetch 是一个基于 ROS2 的多机器人协作抓取系统，集成了自主探索、视觉感知、任务调度、导航和抓取功能。系统支持全自动任务执行：从自动探索环境、目标检测、任务生成到抓取搬运的完整闭环。

### 1.2 核心特性
- ✅ **自主探索** - 双机器人协作探索算法
- ✅ **视觉感知** - 基于颜色和形状的目标检测（红、绿、蓝）
- ✅ **自动任务调度** - 全自动任务生成与分配
- ✅ **智能导航** - 纯追踪导航算法
- ✅ **多机器人协作** - 两个机器人同时工作
- ✅ **Gazebo 仿真** - 完整仿真环境（10m x 10m 室内场景）

### 1.3 技术要求
- Ubuntu 24.04
- ROS2 Jazzy

## 2. 模块和总体功能

### 2.1 cofetch_msgs
**总体功能**：定义系统通信的标准化接口，为所有模块提供消息、服务和动作定义。

### 2.2 cofetch_description
**总体功能**：提供机器人和环境的模型定义，为仿真和可视化提供基础。

### 2.3 cofetch_gazebo
**总体功能**：创建和管理 Gazebo 仿真环境，提供传感器数据和接收控制命令。

### 2.4 cofetch_perception
**总体功能**：实现视觉目标检测和识别，处理相机图像并发布检测结果。

### 2.5 cofetch_exploration
**总体功能**：实现自主环境探索，包括地图构建和前沿导航。

### 2.6 cofetch_scheduler
**总体功能**：实现任务生成和分配，管理多机器人协作任务。

### 2.7 cofetch_manipulation
**总体功能**：实现目标抓取和操作，执行拾取和放置任务。

### 2.8 cofetch_bringup
**总体功能**：启动和协调所有系统组件，提供系统级控制和监控。

### 2.9 cofetch_navigation
**总体功能**：实现纯追踪导航算法，处理机器人路径规划和避障。

### 2.10 cofetch_ui
**总体功能**：提供图形化用户界面，显示机器人状态、任务信息、地图和目标信息。

### 2.11 cofetch_monitoring
**总体功能**：集中管理系统日志，提供实时监控和故障诊断。

### 2.12 cofetch_config
**总体功能**：集中管理系统配置参数，支持动态配置更新。

### 2.13 cofetch_testing
**总体功能**：提供统一的测试框架，包括单元测试、集成测试和端到端测试。

### 2.14 cofetch_docs
**总体功能**：管理系统文档，提供自动文档生成工具。

### 2.15 cofetch_deps
**总体功能**：管理系统依赖，确保版本兼容性并提供依赖安装工具。

### 2.16 cofetch_performance
**总体功能**：分析系统性能瓶颈，提供优化建议。

## 4. 分步骤实现方式

### 4.1 cofetch_msgs

#### 4.1.1 消息类型定义实现
- **方法**：使用 ROS2 消息定义语法
- **实现步骤**：
  1. 创建 msg 目录
  2. 编写 ObjectInfo.msg、ObjectList.msg、TaskInfo.msg、TaskList.msg、RobotStatus.msg、FrontierInfo.msg、PathInfo.msg、NavStatus.msg、SystemStatus.msg、LogMsg.msg
  3. 更新 CMakeLists.txt 和 package.xml
  4. 构建生成消息代码
- **产出文件**：
  - msg/*.msg 文件
  - 自动生成的消息代码

#### 4.1.2 服务类型定义实现
- **方法**：使用 ROS2 服务定义语法
- **实现步骤**：
  1. 创建 srv 目录
  2. 编写 GetRobotStatus.srv、GetTaskStatus.srv、GetConfig.srv、SetConfig.srv、GetSystemStatus.srv
  3. 更新 CMakeLists.txt 和 package.xml
  4. 构建生成服务代码
- **产出文件**：
  - srv/*.srv 文件
  - 自动生成的服务代码

#### 4.1.3 动作类型定义实现
- **方法**：使用 ROS2 动作定义语法
- **实现步骤**：
  1. 创建 action 目录
  2. 编写 Explore.action、PickObject.action、PlaceObject.action、NavigateTo.action
  3. 更新 CMakeLists.txt 和 package.xml
  4. 构建生成动作代码
- **产出文件**：
  - action/*.action 文件
  - 自动生成的动作代码

### 4.2 cofetch_description

#### 4.2.1 机器人 URDF 模型实现
- **方法**：使用 URDF 语法定义机器人模型
- **实现步骤**：
  1. 创建 urdf 目录
  2. 选择官方底盘和机械臂模型（如 TurtleBot4 底盘 + MoveIt 兼容机械臂）
  3. 编写机器人 URDF 文件，集成官方模型

  1. 创建 worlds 目录

  3. 添加墙壁、障碍物等元素
  4. 测试环境加载
- **产出文件**：
  - worlds/indoor.world
  - 环境配置文件

#### 4.2.3 机器人网格和材质实现
- **方法**：使用 3D 建模工具创建网格，定义材质

  1. 创建 meshes 目录

### 7.1 底盘选型
- **推荐**：TurtleBot4
  - **理由**：ROS2 官方支持，集成激光雷达和相机，适合室内导航
  - **特点**：模块化设计，易于扩展，性能稳定

### 7.2 机械臂选型
- **推荐**：Universal Robots UR3e
  - **理由**：轻量级，精度高，ROS2 支持良好
  - **特点**：负载 3kg，工作半径 500mm，适合抓取小型物体

### 7.3 传感器配置
- **相机**：Intel RealSense D435i
  - **理由**：深度相机，同时提供 RGB 和深度信息
  - **特点**：适合目标检测和抓取规划

- **激光雷达**：SICK TIM571
  - **理由**：高精度，可靠性高，适合室内环境
  - **特点**：扫描范围 270°，距离 20m，适合导航和避障
